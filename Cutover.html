Element 3: Runbook Automation using Cutover

Big Bank is interested in understanding how they can speed up the migration by using automations.

Cutover is a SaaS solution that standardizes and automates technology operations processes such as IT and cyber disaster recovery, release management, and cloud migration. It bridges the gap between teams and technology to increase efficiency and reduce risk with automated runbooks. It can integrate with any other solution via APIs to manage processes end to end.

Rapid Migration with Hyper Automation, is a solution built by AWS ProServe that integrates Cutover with other AWS tools and services to perform automated migrations. It includes an automation library with a set of common rehost patterns that can be customized and extended based on the customer needs.

RMHA Architecture overview:
In the Cutover instance that you will use for the demo, we have included a sample runbook template for a simplified rehost migration. Due to the time constraints of the event, the tasks from Cutover will not perform the actual migration of servers but just do a connection. The actions performed are controlled by the parameters sent in the custom fields (Automation ID) (Not required for this demo, but further information can be found in the delivery kit)

 

To prepare the demo follow these steps:

Step 1: Access Cutover and visualize the pre-configured Rehost template

  1. Access to the cutover instance (access details provided via Email)
  2. Accept invite and create your password. Login with credential just created
  3. Click on the existing workspace called “Migratapalooza Group X”

4. Go to Templates

5. Select the template for Rehost Migration

6. Visualize Template tasks

Each item in the template is a runbook task, the icon on the left of the task indicates the task type, as follows:

  • Circle shape: a manual task. It requires the user to open/close the task.
  • Gear shape: an automated task. It triggers an automation (in this case connecting to an AWS account) and it will start/close automatically in the Cutover.com runbook.
  • Diamond shape: a milestone. These milestones will also be tracked in the Cutover.com dashboards. They require a manual input (user click) for the runbook to move past the milestone step.
  • Triangle shape: a decision point or branching. These tasks require the user to choose a path in the runbook based on a decision (e.g., go/no-go final migration decision)


The template that you will find pre-created in cutover is designed to do a Rehost migration from servers an on-premises data center to EC2 Instances on AWS. It contains the following tasks (note the description is provided for context and clarity, but the provided runbook for the demo will not perform the actual tasks due to time constraints)

  • Import Wave Data will automatically load migration metadata from an input file into the CMF metadata store.
  • Install Replication Agents will use a CMF automation to install and configure the MGN agent on each server in this wave, using the supplied server credentials.
  • Monitor Data Replication 1 will continuously monitor Replication Agent data replication from the on-premises servers to the target Replication Agent replication servers until the copy reaches 100%.
  • Ready For Testing this is a milestone task, this action needs to be ticked manually and will enable the next set of automated tasks, which will execute without human intervention.
  • Validate Launch Template will use a CMF automation to verify that the AWS Replication Agent launch templates are correct based on the target metadata supplied. For example, the supplied subnets, security groups, and other settings.
  • Launch Test Instances will use a CMF automation to signal AWS Replication Agent to launch a set of test instances in the target subnets, based on the latest snapshot taken. 
  • Verify Test Instances will use a CMF automation to determine if all test servers have been converted to EC2 successfully, have booted up and are responsive. Amazon EC2 service has a mechanism to determine if the OS is running and responsive. This is called Instance Status Check, which is leveraged by the automation to determine the instance status.
  • Revert Instances status to Ready for Testing, this CMF automation will revert the status of the Replication Agent service to Ready for Testing. Doing so permits to run further tests or move the next phase of cutover.
  • Terminate Test Instances will use a CMF automation to terminate the recently launched (and verified) test instances.
  • Copy post-launch scripts will copy scripts stored in the CMF Automation server to the source servers. These scripts will be placed in an Replication Agent folder and will be automatically executed by Replication Agent when the target EC2 instance starts for the first time post-cutover.
  • Monitor Data Replication 2 will continue to monitor data replication from the simulated on-premises servers to the target AWS environment. This will ensure that the continuous data replication is at 100% before moving to the next phase of cutover.
  • Mark Ready for Cutover will use a CMF automation to change the Replication Agent status of the servers, signaling that they are ready to be migrated completely.
  • Shutdown All Servers will use a CMF automation to shutdown all on-premises servers in the wave, this is in order to ensure a consistent copy of the server in AWS and prevent further changes.
  • Launch Cutover Instances will use a CMF automation to signal Replication Agent that cutover instances (i.e., the final copy of the servers) should now be launched in EC2.
  • Ready For UAT this is a milestone task, this action needs to be ticked manually.
  • Perform UAT is a task for manual verification that the migrated servers are accessible. It needs to be ticked manually after testing has been completed.
  • Go Live? branching (or decision) runbook task. You will have to decide which path to abandon. If the testing was successful, proceed to abandon the NO: Rollback path. Otherwise, proceed to abandon the YES: Go live path, as shown below.
  • Go Live. This path will trigger an automation to finalize the migration in AWS MGN, which will conclude the process.
  • End milestone, proceed to to open the AWS Replication Agent console in the workshop AWS Account. Verify the servers status is Cutover complete

 

Step 2: Create and execute a runbook based on the existing template

  1. Go back to templates overview. Click on the create button at the right to create a new runbook based on the template:
  2. Start Runbook
  3. Execute manual start tasks
  4. Automatic tasks for migration steps will start executing
  5. On breakpoints for manual check, confirm execution
  6. When reaching the task for UAT testing, wait ~ 10 minutes before completing it. This waiting time is meant to represent the time spent in performing the actual test of the migrated workloads
  7. In the Go Live , select abandon for the rollback tasks and continue for the Go Live
  8. Finalize the run-book execution
  9. Go to the post implementation dashbaord and review results
